# Gemba

*Large-scale side-of-the-desk projects: big impact, minimal cognitive load.*

In lean manufacturing, gemba (現場) is "the actual place" — the factory floor, where real work happens. A gemba walk is when leadership observes the work directly, not through reports, and leaves actionable feedback as they go. Gemba the product is built around that metaphor. Operators running agentic software projects today hit a consistent failure pattern: planning is disconnected from execution, priority is invisible, expertise is siloed, state is fragile, and the orchestration tooling is tied to one runtime. Gemba is the browser-based UI for walking the floor of an agentic project — seeing the work at the right grain, directing it, with a roster of configurable LLM specialists a click away for the expertise the operator doesn't have time to personally deliver for every decision.

The shape is a single Go binary with an embedded React SPA. It pairs exactly one Work Coordination Plane adaptor (the issue tracker — Beads, Jira, Linear, GitHub Projects, Azure DevOps, …) with exactly one Agent Orchestration Plane adaptor (the agent runtime — Gas Town, Gas City, LangGraph, OpenHands, Devin, …) under a typed contract. The UI is adaptor-agnostic: no backend vocabulary leaks in; what renders is what the active capability manifests declare. Beads and Gas Town ship as the v1 reference pack-ins. Jira and LangGraph are the forcing-function adaptors that prove the abstraction isn't Beads- or Gas-Town-shaped — if the contract handles Jira's workflow FSM, Linear is easy; if it handles LangGraph's checkpoint-and-graph topology, OpenHands is easy.

The expertise layer is a first-class Persona framework — two varieties. Coaches are conversational and advisory: Architect, Code Reviewer, UX Expert, Technical Writer, Documentarian, Onboarder. Managers are agentic and can be gate-blocking: Project Manager, QA, Deployment Engineer, DevOps/SRE, Security. Managers execute scoped tasks directly — PM's `add_to_backlog` skill files beads while consulting the Architect and QA mid-task; QA's gate can block a release; every persona has separately configurable prompt, context providers, and output validation. Jam Sessions formalize the interactive multi-topic decisioning conversation the operator and PM fall into naturally: the PM aggregates escalations from every worker (polecat HITL, Manager persona suspension, gate failures, budget crossings, adaptor-degraded events) into one agenda and walks the operator through ratification — resumable, auditable, and the native mode for working a busy backlog at scale.

The home screen is an Epic-granular Kanban. Cards are Epics (not individual stories), with readiness counts, parallel-group membership, escalation state, and token-budget posture; drill-down reveals member work items. The Plan view lets the operator drag to reorder, toggle in or out of scope, and launch the batch after parallel-safety is verified. Project values (innovation, transparency, execution, empathy) inject at the top prompt layer for every persona invocation; guardrails are values with Manager-enforced blockers; project and epic goals flow into relevant prompts; workspace modes (unsupervised / supervised / managed) govern how much auto-approval is on the table. Checkpoints atomically snapshot every git repo involved, the Beads database, live session context summaries, sidecar state, and artifacts — operators can "undo back to yesterday" and roll every dimension back together. Agentic code analysis — GitNexus-class knowledge graphs, pluggable backends — is a first-class capability that feeds personas' context and powers richer source-code-repo bootstrap.

Extensibility is three surfaces. Adaptors implement the typed two-plane contract and run the importable conformance harness in their own CI. Personas arrive as TOML files. Packs bundle personas + skills + default values + guardrails + adaptor preferences — arbitrary in name and intent, not constrained to executive roles (CTO today; CEO, COO, CRO/LRC as seed future packs; domain packs like fintech-compliance or healthcare-hipaa; task packs like migration or launch-readiness; customer-specific private packs), signed for provenance, public or private. Bootstrap pulls from Jira, Beads, an existing source-code repo (the agentic code analysis walks module structure, call graph, and entry points to propose candidate Epic decomposition), or fresh — validates the generated description, goals, values, guardrails, and starting plan for internal consistency before the operator ratifies. XRay and configurable linters integrate at the organization surface as QA's canonical evidence and style tooling. Gemba core is open-source; commercial value accrues to specialty pack curation and to training and change-management consulting for teams adopting agentic workflows at scale.

The factory floor, the walk, and the specialists — all in one pane of glass, at the right grain, with nothing else to switch between.
